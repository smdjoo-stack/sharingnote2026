# 📌 랜딩페이지 + GitHub 폴더 기반 자동 아카이브 시스템 계획서

---

## 1. 프로젝트 개요

### 1) 목적
- GitHub에 업로드한 폴더 내용을
- 자동으로 최신순 정렬하여
- 랜딩페이지에서 아카이브 형식으로 보여준다
- 모바일에서도 최적화된 UI로 사용 가능하게 한다

---

## 2. 전체 폴더 구조

```
project-root/
│
├── index.html          → 랜딩페이지
├── archive/            → 날짜별 콘텐츠 폴더
│   ├── 2026-02-20-post1/
│   │   └── index.html
│   ├── 2026-02-18-post2/
│   │   └── index.html
│   └── 2026-02-15-post3/
│       └── index.html
│
├── assets/
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── main.js
│
└── data/
    └── posts.json      → 게시물 목록 관리용
```

---

## 3. 구현 방식 (JSON 기반 자동 아카이브 - 추천)

### 작동 원리
1. archive 폴더에 콘텐츠 추가
2. posts.json에 메타데이터 추가
3. JavaScript가 최신 날짜 기준으로 자동 정렬
4. 랜딩페이지에 자동 렌더링

---

## 4. posts.json 예시

```json
[
  {
    "title": "설교 아카이브 1",

    "path": "archive/2026-02-20-post1/index.html",
    "description": "하나님이 나를 발견하셨다"
  },
  {
    "title": "설교 아카이브 2",

    "path": "archive/2026-02-18-post2/index.html",
    "description": "넓은 곳으로 이끄시는 음성"
  }
]
```

---

## 5. 최신순 정렬 JS 코드 (main.js)

```javascript
fetch('data/posts.json')
  .then(response => response.json())
  .then(posts => {
    posts.sort((a, b) => new Date(b.date) - new Date(a.date));

    const container = document.getElementById('archive');

    posts.forEach(post => {
      const item = `
        <div class="post">
          <h2>${post.title}</h2>
          <p>${post.date}</p>
          <p>${post.description}</p>
          <a href="${post.path}">보러가기</a>
        </div>
      `;
      container.innerHTML += item;
    });
  });
```

## 8. GitHub Pages 배포 방법

1. GitHub Repository 생성
2. 위 폴더 구조 그대로 업로드
3. Settings → Pages → main 브랜치 선택
4. 저장 후 자동 배포 URL 생성
